# Reconstruction_of_the_solar_spectrum
Описание задачи
  Восстановления спектра солнечного излучения по ограниченному набору мультиспектральных измерений.
  Входные данные: 9 значений интенсивности на фиксированных длинах волн (415, 445, 480, 515, 555, 590, 630, 680, 912 нм)
  Выходные данные: Полный спектр из 1904 точек в диапазоне 400–900 нм

Файлы и настраиваемые параметры

architecture/ — Архитектуры моделей
  model_1.py — SpectralGAN
    Генеративно-состязательная сеть. Генератор восстанавливает спектр из 9 входных значений, дискриминатор отличает реальные спектры от поддельных.
    Параметры генератора:
      hidden_dim=512 — размер скрытого слоя
      n_blocks=4 — количество residual блоков
      dropout=0.1 — вероятность дропаута
    Параметры дискриминатора:
      hidden_channels=64 — начальное количество каналов свертки
      n_layers=3 — количество сверточных слоев

  model_2.py — SpectralUNet
    U-Net архитектура с SE блоками (squeeze-excitation). Использует encoder-decoder с skip-connections для сохранения деталей.
    Параметры:
      base_channels=32 — базовое количество каналов, удваивается на каждом уровне encoder

  model_3.py — SpectralTransformer
    Transformer encoder с positional encoding. Вход проецируется в латентную последовательность, обрабатывается трансформером, затем апсемплится до полного спектра.
    Параметры:
      d_model=64 — размерность модели
      num_heads=4 — количество голов внимания
      num_layers=2 — количество слоев трансформера
      ff_dim=128 — размерность feed-forward слоя
      dropout=0.1 — вероятность дропаута
      latent_len=128 — длина латентной последовательности

  model_4.py — LightSpectralMamba
    Легковесная State Space Model (Mamba) с линейной сложностью. Использует свертку с группировкой и SiLU активацию.
    Параметры:
      d_model=128 — размерность модели
      n_layers=6 — количество слоев Mamba

  model_5_cascade.py — SpectralResNet
    Каскадный ResNet с блоком рефайнинга. Сначала генерирует грубую аппроксимацию, затем уточняет детали.
    Параметры:
      hidden_dim=512 — размер скрытого слоя
      latent_len=32 — длина латентной последовательности
      dropout=0.1 — вероятность дропаута

предсказание/ — Обучение и предсказание

  models.py — Grid Search
    Перебирает все комбинации моделей, методов предобработки и функций потерь. Обучает каждую комбинацию, сохраняет метрики и строит графики.
    Параметры:
      модели — SpectralUNet, SpectralTransformer, LightSpectralMamba, SpectralResNet (какие модели участвуют в переборе)
      x_methods — standart, raw (методы предобработки входных данных: стандартизация или без изменений)
      y_methods — robust, standart (методы предобработки целевых спектров)
      функции потерь — Huber_delta0.5, MSE+L1 (функции ошибки для оптимизации)
      эпохи: 180 — количество полных проходов по обучающим данным
      lr: 1e-3 — скорость обучения (learning rate)
      batch_size: 32 — размер мини-батча
      test_size: 0.2 — доля данных для тестовой выборки
      val_size: 0.2 — доля данных для валидационной выборки
      random_state: 42 — seed для воспроизводимости результатов
      print_every: 5 — частота вывода информации в консоль (каждые N эпох)

  pca_mlp.py — PCA + MLP
    Базовое решение. Уменьшает размерность спектра через PCA, обучает MLP предсказывать коэффициенты, затем восстанавливает спектр обратным преобразованием.
    Параметры:
      n_components: 5 — количество PCA компонент (размерность латентного пространства)
      hidden_layers: [1024, 1024, 1024] — архитектура MLP: три скрытых слоя по 1024 нейрона
      x_methods — standart, raw (методы предобработки входных данных)
      y_methods — standard, robust (методы предобработки целевых спектров)
      функции потерь — Huber(delta=1.0), Huber(delta=0.5), L1, SmoothL1, MSE+L1
      эпохи: 150 — количество полных проходов по обучающим данным
      lr: 1e-3 — скорость обучения
      batch_size: 64 — размер мини-батча
      weight_decay: 1e-4 — коэффициент L2-регуляризации
      print_every: 10 — частота вывода информации в консоль (каждые N эпох)

  train_gan.py — GAN обучение
    Adversarial обучение генератора и дискриминатора. Использует Hinge loss для GAN и L1 loss для стабильности.
    Параметры:
      x_method: standard — метод предобработки входных данных
      y_method: standard — метод предобработки целевых спектров
      эпохи: 150 — количество полных проходов по обучающим данным
      lr_gen: 1e-4 — скорость обучения генератора
      lr_disc: 1e-4 — скорость обучения дискриминатора
      lambda_l1: 50 — вес L1 потери (чем больше, тем ближе генератор к истинному спектру)
      n_critic: 1 — количество обновлений дискриминатора на одно обновление генератора
      batch_size: 32 — размер мини-батча
      print_every: 5 — частота вывода информации в консоль (каждые N эпох)

работа с датасетом/ — Подготовка данных
  sync_spectra.py — Синхронизация
    Сопоставляет спектры (FSR05_spectr/) и мультиспектры (multi_spectr/) по времени. Создает paired файлы с 4 строками.
    Параметры:
      max_time_diff: 0.5 секунды — максимальная допустимая разница во времени между спектром и мультиспектром для создания пары
      spectral_range: 400–900 нм — диапазон длин волн, который оставляется из полного спектра
      multi_wavelengths: [415, 445, 480, 515, 555, 590, 630, 680, 912] — фиксированные длины волн мультиспектральных каналов

  clean_dataset.py — Очистка
    Удаляет выбросы из датасета по суммарной яркости спектра.
    Параметры:
      low_threshold: 500000 — нижний порог суммарной яркости (образцы с яркостью ниже удаляются)
      high_threshold: 2250000 — верхний порог суммарной яркости (образцы с яркостью выше удаляются)
      use_integral: False — если True, использует численное интегрирование (точнее); если False, использует простую сумму значений

  search_for_duplicates.py — Поиск дубликатов
    Находит файлы с одинаковыми мультиспектральными значениями X и строит графики для сравнения.
    Параметры:
      tolerance: 1e-6 — допустимая погрешность при сравнении X (значения считаются одинаковыми, если отличаются не более чем на tolerance)	

  analyze_spectral_pairs.py — Анализ датасета
    Проводит полный статистический анализ датасета: распределения, PCA, корреляции, строит визуализации.
    Параметры:
      n_channels: 9 — количество спектральных каналов для разбиения
      n_pca_per_channel: 3 — количество PCA компонент для каждого канала
      n_components_pca_full: 10 — количество компонент для полного PCA анализа
      spectral_ranges: [(400,450), (450,500), (500,600), (600,700), (700,800), (800,900)] — диапазоны длин волн для анализа PCA по диапазонам
