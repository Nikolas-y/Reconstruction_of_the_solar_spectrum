# Reconstruction of the Solar Spectrum

Восстановление спектра солнечного излучения по ограниченному набору мультиспектральных измерений.

---

# Описание задачи

## Входные данные

9 значений интенсивности на фиксированных длинах волн:

* 415 нм
* 445 нм
* 480 нм
* 515 нм
* 555 нм
* 590 нм
* 630 нм
* 680 нм
* 912 нм

## Выходные данные

Полный спектр из **1904 точек** в диапазоне:

* **400–900 нм**

# Архитектуры моделей

## `architecture/model_1.py` — SpectralGAN

Генеративно-состязательная сеть.

* Генератор восстанавливает спектр из 9 входных значений
* Дискриминатор отличает реальные спектры от синтетических

## Параметры генератора

| Параметр     | Значение | Описание                   |
| ------------ | -------- | -------------------------- |
| `hidden_dim` | `512`    | Размер скрытого слоя       |
| `n_blocks`   | `4`      | Количество residual-блоков |
| `dropout`    | `0.1`    | Вероятность dropout        |

## Параметры дискриминатора

| Параметр          | Значение | Описание                    |
| ----------------- | -------- | --------------------------- |
| `hidden_channels` | `64`     | Начальное число каналов     |
| `n_layers`        | `3`      | Количество сверточных слоев |

---

## `architecture/model_2.py` — SpectralUNet

U-Net архитектура с SE-блоками (*Squeeze-and-Excitation*).

Использует:

* encoder-decoder структуру
* skip-connections
* сохранение локальных спектральных деталей

## Параметры

| Параметр        | Значение | Описание              |
| --------------- | -------- | --------------------- |
| `base_channels` | `32`     | Базовое число каналов |

---

## `architecture/model_3.py` — SpectralTransformer

Transformer Encoder с positional encoding.

### Особенности

* вход проецируется в латентную последовательность
* последовательность обрабатывается Transformer-блоками
* далее выполняется upsampling до полного спектра

## Параметры

| Параметр     | Значение |
| ------------ | -------- |
| `d_model`    | `64`     |
| `num_heads`  | `4`      |
| `num_layers` | `2`      |
| `ff_dim`     | `128`    |
| `dropout`    | `0.1`    |
| `latent_len` | `128`    |

---

## `architecture/model_4.py` — LightSpectralMamba

Легковесная State Space Model (Mamba).

### Особенности

* линейная сложность
* grouped convolution
* активация SiLU

## Параметры

| Параметр   | Значение |
| ---------- | -------- |
| `d_model`  | `128`    |
| `n_layers` | `6`      |

---

## `architecture/model_5_cascade.py` — SpectralResNet

Каскадный ResNet с блоком refinement.

### Этапы работы

1. Генерация грубого спектра
2. Уточнение деталей

## Параметры

| Параметр     | Значение |
| ------------ | -------- |
| `hidden_dim` | `512`    |
| `latent_len` | `32`     |
| `dropout`    | `0.1`    |

---

# Обучение и предсказание

Перебор комбинаций:

### Используемые модели

* SpectralUNet
* SpectralTransformer
* LightSpectralMamba
* SpectralResNet

### Предобработка

| `print_every`  | `5`      |

## `предсказание/pca_mlp.py` — PCA + MLP

Базовый подход:

1. PCA уменьшает размерность спектра
2. MLP предсказывает PCA-коэффициенты
3. Обратное PCA восстанавливает спектр

## Параметры

### PCA

| Параметр       | Значение |
| -------------- | -------- |
| `n_components` | `5`      |

### Архитектура MLP

```python
[1024, 1024, 1024]
```

### Предобработка

#### Для входов (`X`)

* `standard`
* `raw`

#### Для целевых спектров (`Y`)

* `standard`
* `robust`

### Функции потерь

* `Huber(delta=1.0)`
* `Huber(delta=0.5)`
* `L1`
* `SmoothL1`
* `MSE + L1`

## Параметры обучения

| Параметр       | Значение |
| -------------- | -------- |
| `epochs`       | `150`    |
| `lr`           | `1e-3`   |
| `batch_size`   | `64`     |
| `weight_decay` | `1e-4`   |
| `print_every`  | `10`     |

---

Adversarial-обучение генератора и дискриминатора.

### Используемые loss-функции

* L1 Loss — для стабильности реконструкции

## Параметры

| `lambda_l1`   | `50`       |
| `n_critic`    | `1`        |
| `batch_size`  | `32`       |
| `print_every` | `5`        |

* спектры (`FSR05_spectr/`)

Создает paired-файлы из 4 строк.

| ---------------- | ------------ |
| `max_time_diff`  | `0.5 сек`    |

[415, 445, 480, 515, 555, 590, 630, 680, 912]
